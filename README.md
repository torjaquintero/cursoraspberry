# cursoraspberry

# Ejercicio No. 1

Este ejercicio introduce el control básico de hardware utilizando los pines GPIO de una **Raspberry Pi 5** mediante el lenguaje Python. El programa implementa el clásico ejemplo de **parpadeo de un LED**, configurando un pin GPIO como salida digital y alternando su estado lógico entre encendido y apagado con un intervalo de un segundo. A través de este ejemplo se presentan conceptos fundamentales de programación en sistemas embebidos, como la configuración del modo de numeración de pines, la escritura de niveles lógicos HIGH y LOW, el uso de retardos temporales y el manejo adecuado de interrupciones del usuario. Además, el código incorpora buenas prácticas de desarrollo al incluir manejo de excepciones y liberación de los recursos GPIO al finalizar el programa, proporcionando así una primera aproximación clara a la interacción entre software y hardware en plataformas embebidas basadas en Raspberry Pi.


# Ejercicio No. 2

Este ejercicio implementa un **sistema básico de semáforo vehicular utilizando la Raspberry Pi y la librería RPi.GPIO en Python**. El programa controla tres LEDs conectados a los pines GPIO que representan las señales **verde, amarillo y rojo**, generando una secuencia de estados típica de señalización de tráfico. El objetivo del ejercicio es introducir conceptos fundamentales del diseño de sistemas embebidos, como el **control de múltiples salidas digitales, la implementación de sistemas basados en estados y la gestión de temporización mediante retardos de software**. Este ejercicio forma parte de una serie de prácticas progresivas orientadas al aprendizaje de **programación de hardware con Raspberry Pi y desarrollo de sistemas electrónicos embebidos**.

# Ejercicio No. 3

Este ejercicio introduce la integración entre **control de hardware en Raspberry Pi y visualización mediante una interfaz gráfica en Python**. El programa controla un LED físico conectado al **GPIO17** utilizando la librería `RPi.GPIO`, mientras que una ventana gráfica desarrollada con **Tkinter** muestra un LED virtual cuyo color refleja el estado del dispositivo real. El sistema alterna automáticamente entre encendido y apagado cada segundo, actualizando tanto el hardware como la representación visual. El objetivo del ejercicio es familiarizarse con la creación de **interfaces gráficas simples, el uso del objeto Canvas para representar dispositivos electrónicos y la sincronización entre GPIO y aplicaciones gráficas mediante tareas periódicas con el método `after()`**. Este ejercicio forma parte de una serie progresiva orientada al aprendizaje práctico de **programación de sistemas embebidos con Raspberry Pi**.

# Ejercicio No. 4

Este ejercicio amplía el sistema de semáforo desarrollado previamente incorporando una **interfaz gráfica en Python utilizando la librería Tkinter**. El programa controla tres LEDs físicos conectados a los pines GPIO de la Raspberry Pi que representan las luces **verde, amarillo y rojo**, mientras que una ventana gráfica muestra simultáneamente tres LEDs virtuales que reflejan el estado del hardware en tiempo real. La secuencia del semáforo se implementa mediante una **máquina de estados simple** y se ejecuta de forma temporizada utilizando el método `after()`, lo que permite actualizar tanto los GPIO como la interfaz gráfica sin bloquear la aplicación. Este ejercicio introduce conceptos importantes como la **visualización del estado de un sistema embebido, la sincronización entre hardware y GUI, y la creación de animaciones temporizadas en aplicaciones Python** orientadas al control de dispositivos electrónicos.

# Ejercicio No. 5

Este ejercicio introduce la **lectura de entradas digitales en la Raspberry Pi** mediante la incorporación de un botón que permite generar una solicitud de cruce peatonal en el sistema de semáforo. El programa controla tres LEDs conectados a los pines GPIO que representan las luces **verde, amarillo y rojo**, y además monitorea un pulsador conectado al **GPIO23** configurado con resistencia interna *pull-up*. Cuando el sistema se encuentra en estado **VERDE**, el peatón puede presionar el botón para solicitar el cambio del semáforo; si el botón se presiona durante los estados **AMARILLO o ROJO**, la solicitud es ignorada. El programa también implementa **protección contra rebotes (debounce) por software** para evitar múltiples activaciones del botón. Este ejercicio introduce conceptos fundamentales como **lectura de entradas digitales, manejo de eventos externos, lógica dependiente del estado del sistema y diseño de máquinas de estados simples**, elementos esenciales en el desarrollo de sistemas embebidos interactivos.

# Ejercicio No. 6

En este ejercicio se implementa un **sistema de semáforo vehicular con solicitud peatonal utilizando interrupciones GPIO en la Raspberry Pi**. A diferencia del ejercicio anterior, donde el botón era leído constantemente mediante *polling*, en este programa se emplea un mecanismo de **interrupciones por hardware** que permite detectar automáticamente cuando el botón es presionado. Cuando ocurre este evento, se ejecuta una **rutina de servicio de interrupción (ISR)** encargada de registrar la solicitud peatonal de manera rápida y eficiente. La implementación sigue buenas prácticas de diseño en sistemas embebidos: la ISR es corta, no utiliza retardos y únicamente genera un evento que luego es procesado por el bucle principal del programa. Además, se implementa **debounce por software** para evitar activaciones múltiples del botón. Este ejercicio introduce conceptos fundamentales como **programación reactiva basada en eventos, manejo de interrupciones y diseño de máquinas de estados**, ampliamente utilizados en el desarrollo de sistemas embebidos e IoT.
